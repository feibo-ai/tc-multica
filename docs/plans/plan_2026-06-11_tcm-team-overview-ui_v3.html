目标

Goal
tc-multica 前端三块改造，一个 plan 分 3 波次：① 新增「团队」顶部 tab 总览页——全员默认落地、全员可见全员，每个成员一张高密度信息卡（封闭清单维度，全通俗中文命名 + 关键含义常显，无英文行话），让 boss 一眼看清每个人在干什么、当前状态如何；② 侧边栏「搜索 / 新建issue」从顶部移到底部 + 新增可持久化的收缩功能；③ 项目页主从布局——点 issue 后保留「侧边栏 + 项目栏 + issue详情」三栏，不再整页跳走丢失列表栏。

完成标准

  • 波次1·后端：GET /api/team/overview 端点活、挂 RequireWorkspaceMember 受保护组、所有 query 过 workspace_id，一次返回全员卡数据；【N2】端点 query 数与成员数无关——GROUP BY 一次聚合、handler 内不循环 N 次（落一条 review 检查项）；卡数据含默认排序键（卡住 > 满负荷 > 活跃，后端给）；make sqlc 无 drift；go test ./internal/handler/ 绿；端点配 zod + parseWithFallback + ≥1 喂坏数据测试。
  • 波次1·前端：新路由 /[workspaceSlug]/team；顶部「团队 | 项目」tab；成员卡网格 web + desktop 双端渲染；落 packages/views/team/（core 零 react-dom/next，views 零 next）；【N-D1】视觉与数据复用既有 profile card 体系（member-profile-card 的 ActorAvatar/RoleBadge/min-w-0 模式）+ sparkline.tsx + chart token 堆叠条，不另造；当前用户自己的卡视觉置顶/高亮。
  • 波次1·设计·a11y（B-D2·硬约束）：健康徽章/角色 pill/状态一律 dot(bg-success/warning/destructive token) + text-foreground 文字标签，禁用彩色文字 / 禁用 Badge destructive variant / 禁照抄 squad-profile-card leader chip（均违 case TEA-70）；任务分布堆叠条图例 = 彩点 + 文字标签 + 数字 三件套，「卡住」段用 --warning/--destructive 跳出蓝阶（chart-1..5 是同色相明度阶、色盲不可辨）；无硬编码 Tailwind 色（走 OKLCH token）；零 emoji。
  • 波次1·设计·状态口径（B-D1·硬约束）：头部「当前状态」落 agent 口径——「智能体 N 运行中 / 空闲」（基于 AgentTaskQueue active + squad LastActiveAt 代理），不假装人 online presence（团队无真人 presence 后端，member-profile-card:22 已拒绝伪造）；tooltip 写清「按最近任务派发估算」。
  • 波次1·设计·信息层级（B-D3）：8 指标的单位/含义常显（副标题或常驻小字），hover tooltip 仅作补充、不作唯一载体（触屏可达）；数字块分主次——关键信号（卡住/满负荷/在跑）显著上浮，AI用量/提交数次要弱化，不 4×2 等权平铺。
  • 波次2·侧边栏（B3·复用 shadcn）：搜索 + 新建issue 移到 SidebarFooter 仍可触发；收缩复用既有 SidebarProvider 折叠态（collapsible='icon' + cookie sidebar_state 持久化），dashboard-layout 透传 defaultOpen，不新建 Zustand store；折叠态下「新建issue」保留可识别图标 + tooltip（高频动作不藏死）。
  • 波次3·主从布局（B2·可选回调）：board-card/list-row 加可选 onSelect/onIssueClick 回调、默认 fallback AppLink；仅 project-detail 传 select → 受控 selectedIssueId（store + ?issue= URL 同步）→ 内嵌 IssueDetail（复用 onDelete/onDone）；选中行有高亮、有显式返回入口（关闭/Esc）；my-issues/issues-page/actor-issues-panel 三共享面导航不受影响；深链 /issues/[id] 仍全屏直达；desktop 经 URL param 同步对齐 Route categories/Tab isolation。
  • 全程：make check 全绿；原子 Conventional 提交；每段 AI diff commit 前过人眼；严守包边界（页放 views 双端挂载不复制、UI 用 packages/ui Base UI/shadcn）。
